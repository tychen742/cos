# AI Guidelines

Use this file as the project-local guide for AI-assisted work on the C book.

## Read First

1. Read `README.md` for the repository map.
2. Read `authoring/BOOK_PLAN.md` for audience, scope, and chapter sequence.
3. Read `authoring/STYLE_GUIDE.md` and `authoring/QUALITY_CHECKLIST.md` before drafting or reviewing teaching material.
4. Read the target chapter's `MATERIALS.md` and `ORGANIZATION.md` before changing chapter examples, prose, or exercises.

## Working Rules

- Keep book-wide planning in `authoring/`.
- Keep chapter planning beside chapter material under `chapters/`.
- Keep runnable teaching examples focused on one main idea early in the book.
- Verify C examples when feasible and report the command used.
- Record deferrals in planning files instead of expanding a chapter sideways.

## Reusable Skill

The reusable Codex skill for this workflow is `book-authoring`. Project-specific decisions in this folder take precedence over shared skill defaults.
