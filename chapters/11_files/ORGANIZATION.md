# Chapter 11 Organization: Files

## Chapter Goal

Read and write data outside the running program.

## Learning Outcomes

- Open, read or write, and close a file in bounded examples.
- Handle a failed file operation.
- Explain where sample input belongs in the chapter materials.

## Sequence

1. File motivation and paths.
2. Reading a file.
3. Writing a file.
4. Errors and cleanup.

## Systems Lab

Read project input and write project output with explicit error paths, then choose whether a bounded file-descriptor preview belongs beside the standard-stream explanation.

## Deliberately Deferred

- operating-system API depth unless the chapter explicitly chooses that path.
