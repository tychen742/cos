# Chapter 04 Organization: Conditionals

## Chapter Goal

Turn expressions into decisions.

## Learning Outcomes

- Write `if`, `else if`, and `else` branches.
- Use a `switch` when cases fit that form.
- Recognize when a ternary expression stays readable.

## Sequence

1. Conditions and branch selection.
2. Multi-branch decisions.
3. Ternary expression.
4. `switch`.

## Systems Lab

Add bounded input validation with a failure path that returns a nonzero exit status and reports a problem on standard error, preparing command-line project behavior.

## Deliberately Deferred

- randomness until the branch examples are clear;
- complex validation flows.
