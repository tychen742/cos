# Chapter 12 Materials: Projects

## Current Material

- No project selected yet.

## Candidate Additions

- multi-file program with headers;
- separate compilation, compilation units, and linking;
- static versus shared libraries at a conceptual level;
- build commands and warning flags;
- build flags, reproducible compile commands, and a minimal Makefile or equivalent workflow;
- packaging and reproducibility notes;
- a dependency that can be built or swapped;
- build-failure cases involving missing headers, link failures, library versions, architecture mismatches, or runtime lookup problems;
- tests or verification steps;
- one capstone that reuses arrays, functions, structs, pointers, files, and memory selectively.

## Final Project Families

- text and data processing tool that reads CSV-like input, computes summaries, and writes output;
- tiny numerical kernel that uses matrix or vector operations to discuss timing, memory layout, and compiler flags;
- C library plus Python caller that makes a native library boundary visible to data or AI workflows;
- dependency-aware CLI that uses a small external library and documents build, link, and runtime failure modes.

## Common Mistakes

- making the project too large for the concepts already taught;
- mixing build-system complexity with the first multi-file explanation.
