# Chapter 09 Organization: Structs and Enums

## Chapter Goal

Model related data with user-defined types.

## Learning Outcomes

- Define and use a simple `struct`.
- Use an `enum` for a small set of named values.
- Recognize when a user-defined type improves a program.

## Sequence

1. Why related fields need a model.
2. Struct definition and field access.
3. Enum values and readable states.
4. Short design example.

## Deliberately Deferred

- full ABI and layout discussion;
- large data-model design patterns.
