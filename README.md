# C for Computational Systems

Programming, memory, operating-system interfaces, and reproducible native software.

The teaching examples are organized as book chapters. Each chapter starts with small runnable programs and can grow toward a chapter project or exercises.

| Chapter | Folder | Focus |
| --- | --- | --- |
| 1 | `chapters/01_intro` | Compiling, `main`, comments, output, input |
| 2 | `chapters/02_variables_types` | Values, variables, primitive types, formatting, conversion |
| 3 | `chapters/03_operators` | Arithmetic, assignment, comparison, logical, bitwise operators |
| 4 | `chapters/04_conditionals` | `if`, `else`, ternary expressions, `switch` |
| 5 | `chapters/05_loops` | `while`, `do while`, `for`, nested loops |
| 6 | `chapters/06_arrays_strings` | Arrays, indexes, traversal, strings |
| 7 | `chapters/07_functions` | Declarations, parameters, return values, recursion |
| 8 | `chapters/08_pointers` | Addresses, dereferencing, pointers with arrays |
| 9 | `chapters/09_structs_enums` | User-defined data with structs and enums |
| 10 | `chapters/10_dynamic_memory` | Allocation, ownership, cleanup |
| 11 | `chapters/11_files` | File I/O and error handling |
| 12 | `chapters/12_projects` | Multi-file programs and capstone work |

## Drafting Notes

- Keep early examples focused on one main idea before adding richer examples.
- Prefer source examples in the chapter folders. Generated binaries and scratch experiments should stay out of the chapter sequence.
- Add exercises after the example progression for a chapter is stable.

## Authoring Files

- `authoring/BOOK_PLAN.md` defines the audience, book promise, chapter sequence, and scope.
- `authoring/PREFACE.md` states the systems perspective and why the book matters now.
- `authoring/STYLE_GUIDE.md` defines writing and C example conventions.
- `authoring/QUALITY_CHECKLIST.md` defines the review bar for chapters and examples.
- `authoring/PROGRESS.md` tracks planning and draft status chapter by chapter.
- `authoring/AI_GUIDELINES.md` tells AI tools how to use this project's authoring files.
- Each chapter folder has `MATERIALS.md` for content inventory and `ORGANIZATION.md` for the chapter contract and sequence.
